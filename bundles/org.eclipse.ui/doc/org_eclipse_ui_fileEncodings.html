File Encodings

Identifier: org.eclipse.ui.fileEncodings

Description: This extension point is used to specify file encodings available to editors that read the contents of a file using a user specified file encoding. Eclipse supplies default definitions for ASCII, UTF-8, UTF-16, UTF-16 big endian, UTF-16 little endian, GBK (Chinese only) and GB18030 (Chinese(PRC) only).

File encodings by default are displayed in all locales but it is possible to restrict the locales they are available in by using the locale key.

The file encodings also specify details for use in building selection actions for them in the workbench. They are label, description, helpContextID and toolTip. The value of an encoding is the string that will be sent to the IEncodingSupport for the workbench.

Configuration Markup:

   <!ELEMENT fileEncoding >
   <!ATTLIST fileEncoding
     id              CDATA #REQUIRED
     label            CDATA #REQUIRED
     value            CDATA #REQUIRED
     helpContextID    CDATA #OPTIONAL
     description      CDATA #OPTIONAL
     tooltip          CDATA #OPTIONAL
     locale           CDATA #OPTIONAL
>

  • id - a unique identifier that can be used to reference this contribution.
  • label - the label that will be used to display the encoding
  • value - the system encoding value in the format as would be returned by System.getProperty("file.encoding")
  • helpContextId - the id of the help entry for this encoding
  • description - a String to be used as the description of this encoding
  • toolTip - a String to be used for a tooltip about the encoding
  • locale - the locale this definition is restricted to. The locale is either specified fully (i.e. the supplied GB18030 definition defines its locale as zh_CW for Simplified Chinese) or for a language code only (i.e. the supplied GBK definition defines its locale as zh which is all Chinese language locales)

Predefined File Encodings

The following encodings are provided by Eclipse:

  • US-ASCII
  • ISO8859-1 (Latin 1)
  • UTF-8
  • UTF-16
  • UTF-16 (big endian)
  • UTF-16 (little endian)
  • GB18030 (Simplified Chinese only)
  • GBK (Chinese only)

Known Problems with Encoding Detection

The value 'Text file encoding' displayed in the Preferences dialog under 'Editors' may be wrong on platforms running Windows 2000/XP when user locale and system locale differ.

Example of the manifestation of the bug: A Japanese user using Japanese Windows 2000 works in New York, United States. The user has selected English (United States) as the user locale. The value 'Text file encoding' displayed by Eclipse is incorrect: 'Cp1252' (English). It should display the system locale 'MS932' (Japanese).

Workaround The user can modify the user locale so that user locale and system locale are identical. In the example above, this means the user should set Japanese as the user locale. Restart Eclipse to see the results of this change.  In the above example the value 'Text file encoding' will be correct: 'MS932' (Japanese) after restart.

Checking the system locale

  • Windows 2000 : Open the Control Panel. Go to Regional Options. Look up the items in the tab 'General', inside the 'Language settings for the system' group. The system locale is the item marked as (Default).
  • Windows XP: Open the Control Panel. Go to Regional and Language Options. Switch to the tab 'Advanced'. The system locale is specified in 'Language for non-Unicode programs'.

Changing the user locale

  • Windows 2000 : Open the Control Panel. Go to Regional Options. The user locale can be modified by changing the location in 'Settings for the current user'.
  • Windows XP: Open the Control Panel. Go to Regional and Language Options. The user locale can be modified by changing the language in 'Standards and formats'.
